로봇의 Flange to TCP 관계를 정의한다.

Parameters
[in]robot_idclink_robot_create() 를 통해 반환받은 로봇 인스턴스 ID
[in]pos_xFlange to TCP position X[mm]
[in]pos_yFlange to TCP position Y[mm]
[in]pos_zFlange to TCP position Z[mm]
[in]ort_xFlange to TCP orientation X[degree]
[in]ort_yFlange to TCP orientation Y[degree]
[in]ort_zFlange to TCP orientation Z[degree]
Remarks
Flange to TCP 관계라 함은 Flange 로부터
TCP 종단까지의 거리 및 방향을 의미한다.
본 함수를 통해 TCP 를 별도로 셋팅하지 않은 경우
TCP 의 위치는 Flange 와 동일한 것으로 간주한다.
즉, 모든 TCP 관련 함수는 Flange 의 위치를
TCP 의 위치로 간주한다.
TCP 설정 후, TCP 를 삭제하고 싶은 경우에는
(즉, 다시 Flange 의 위치를 TCP 의 위치로 사용하고 싶은 경우)
모든 parameter (즉, pos_x ... pos_z, ort_x ... ort_z) 를
0 으로 설정한다.
본 함수의 모든 데이터는 Base 좌표계 기준이다.
(즉, TCP 좌표계 아님)
Returns
returnable error code

const clink_float_t *  p_ort_array_z,
clink_float_t *  p_tool_pos 
)

Flange 끝에 달리는 TCP 의 길이 성분을
여러 각도에서의 pose 데이터를 입력하여
계산 및 적용한다. (3개 이상 pose 데이터 입력 필수)

Parameters
[in]robot_idclink_robot_create() 를 통해 반환받은 로봇 인스턴스 ID
[in]input_sizep_pos_array_x ... p_pos_array_z 및
p_ort_array_x ... p_ort_array_z 의 배열 중
valid 한 데이터의 사이즈
(즉, TCP 길이 및 방향 계산에 사용할 pose 의 개수)
[in]p_pos_array_xPose 데이터 중 위치의 X 성분 배열
[in]p_pos_array_yPose 데이터 중 위치의 Y 성분 배열
[in]p_pos_array_zPose 데이터 중 위치의 Z 성분 배열
[in]p_ort_array_xPose 데이터 중 방향의 X 성분 배열
[in]p_ort_array_yPose 데이터 중 방향의 Y 성분 배열
[in]p_ort_array_zPose 데이터 중 방향의 Z 성분 배열
[out]p_tool_pos계산 결과 tcp from flange position 성분
Remarks
여기서의 pose 데이터는 명확히 말하면
Flange 의 Base 좌표계 기준의 pose 이다.
즉, pose 데이터를 취득할 때,
clink_robot_tcp_from_flange_set() 함수를 통해
모든 position 및 orientation 을 0 으로 설정한다.
이 때는 Flange 와 TCP 는 동일시 된다.
이후, 다양한 방향으로 로봇을 위치시킨 후,
clink_robot_tcp_from_flange_get() 를 통해
데이터를 취득한다.
Returns
returnable error code

CLINK_EXPORT CLINK_API_RESULT clink_robot_tcp_from_flange_multiple_orientation_set ( uint32_t  robot_id,
clink_float_t  ort_x,

)

Flange 끝에 달리는 TCP 의 방향 성분을
미리 지정되어있는 고정된 좌표계와 일치 시킨 후
적용한다.

Parameters
[in]robot_idclink_robot_create() 를 통해 반환받은 로봇 인스턴스 ID
[in]ort_xBase 좌표계 기준 방향 X 성분
[in]ort_yBase 좌표계 기준 방향 Y 성분
[in]ort_zBase 좌표계 기준 방향 Z 성분
[out]p_tool_ort계산 결과 tcp from flange orientation 성분
Remarks
고정 좌표계는 다음과 같다.
Base 좌표계 기준으로 -z축(즉, 아래방향)으로 TCP의 x축 정렬
Base 좌표계 기준으로 x축(즉, 앞쪽방향)으로 TCP의 z축 정렬
(y축은 x,z 축에 의해 결정)
해당 API 적용 전, clink_robot_tcp_from_flange_set() 함수를 통해
모든 orientation 을 0 으로 설정한다.
clink_robot_tcp_from_flange_get() 를 통해
계산된 데이터를 취득 가능하다.
Returns
returnable error code
