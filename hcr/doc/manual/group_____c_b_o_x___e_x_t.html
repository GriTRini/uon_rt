CLINK_EXPORT CLINK_API_RESULT clink_cbox_ext_rdm_start ( const char_t *  p_my_ip,
uint16_t  my_port,
const char_t *  p_dest_ip,
uint16_t  dest_port,
uint32_t  hz_in_sec 
)

UDP 를 활용하여 설정된 목적지로 CBOX 의 데이터 (예: CBOX 의 IO) 및
연결된 모든 로봇의 기본적인 정보를 전송한다. (Realtime Data Monitoring)

Parameters
[in]p_my_ip현재 이 API 를 사용하는 PC 의 NIC 의 IP 주소
[in]my_portUDP 소켓에 사용할 포트 번호. 0 을 입력하는 경우 임의로 할당
[in]p_dest_ip목적지 PC 의 NIC 의 IP 주소
[in]dest_port목적지 PC 의 포트 번호
[in]hz_in_sec1초에 UDP 패킷을 전송할 횟수.
예: 100 인 경우, 100 회를 전송한다.
Remarks
본 기능은 실시간을 보장하지 않는다.
약, 99% 정도의 확률로 100회를 보장한다고 간주하면 된다.
Returns
returnable error code

CLINK_EXPORT CLINK_API_RESULT clink_cbox_ext_robot_rtss_start ( uint32_t  robot_id,
const char_t *  p_dest_ip,
uint16_t  dest_port,
uint32_t  hz_in_sec 
)

UDP/IP 를 활용하여 설정된 목적지로 로봇의 각종 정보를 전달한다. (Real-Time State Streaming)
RDM 기능에 비해 정확한 주기적인 데이터를 제공한다.

Parameters
[in]robot_idclink_robot_create() 를 통해 반환받은 ID 값
[in]p_dest_ip목적지 PC 의 NIC 의 IP 주소
[in]dest_port목적지 PC 의 포트 번호
[in]hz_in_sec1초에 UDP 패킷을 전송할 횟수. 최대 500Hz 까지 지원한다.
Remarks
목적지의 PC에서 UDP 패킷을 수신하기 위해서 아래와 같은 설정이 필요하다.(목적지 PC 설정) Control Box 내 EPC 의 1번 NIC 와 목적지 PC 간 Local Network 구성이 필요하다.
route add 192.168.222.0 mask 255.255.255.0 [Control Box IP] (Windows Command Prompt 에서 입력)
Control Box 의 Default IP 는 192.168.100.200 이다. 즉 변경사항이 없을 시 필요한 기본 설정은 아래와 같다.
ex) route add 192.168.222.0 mask 255.255.255.0 192.168.100.200
해당 기능은 네트워크 구조 상 하나의 Control Box 대상으로만 사용이 가능하다.
2개 이상의 Control Box 접속 상태에서 사용 불가
Returns
returnable error code
